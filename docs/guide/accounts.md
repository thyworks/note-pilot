# アカウント管理 — `/np:account`

note-pilotは**noteアカウントを持っていない人も、持っている人も**同じ環境で管理できます。記事を事前に書き溜めてからアカウントを開設する運用にも対応しています。

## アカウントのステータス

各アカウントには3つのステータスがあります。

| status | 意味 | 記事作成 | 校閲 | 自動投稿 |
|--------|------|---------|------|---------|
| `preparing` | noteアカウント未作成。記事を準備中 | ✅ | ✅ | ❌ |
| `active` | noteアカウントあり。通常運用中 | ✅ | ✅ | ✅ |
| `paused` | 一時停止中 | ✅（確認後） | ✅ | ✅（確認後） |

`preparing` 状態でも記事の作成・校閲は通常通り行えます。noteアカウントがないため自己紹介embedは省略されますが、アカウント開設後にステータスを `active` に変更するだけで、それまでに書いた記事をそのまま投稿できます。

## アカウント一覧を確認する

```
アカウント一覧を見せて
```

```
/np:account list
```

```
アカウントを管理したい
```

表示例:

```
[1] thy-works — Thy works（active）記事35本
[2] fuku-no-tane — 副業のたね（active）記事12本
[3] new-brand — New Brand（preparing）記事0本
```

## 新しいアカウントを追加する

```
アカウントを追加して
```

```
/np:account add
```

対話形式で以下を確認します。

1. アカウント名（表示名）
2. スラッグ（ファイル名に使う英数字識別子）
3. **noteアカウントを持っているか？**
   - **持っていない** → `status: preparing` で作成。note URL は後で設定できます
   - **持っている** → note URL と自己紹介記事URLを入力して `status: active` で作成
4. 扱うテーマ
5. やらないこと（禁止テーマ）

## noteアカウントを開設したら（preparing → active）

noteアカウントを取得したら、ステータスを `active` に切り替えます。

```
アカウントのステータスを変えて
```

```
/np:account status
```

変更時に note URL と自己紹介記事URLの入力を求められます。設定後、それまでに書き溜めた記事は `/np:post` で投稿できるようになります。

## 一時停止・再開する

```
/np:account status
```

`active` → `paused` に変更すると、一時停止の理由を記録できます。
`paused` → `active` に変更すると、再開日が自動で記録されます。

## 複数アカウントを切り替える

複数のアカウント設定ファイルがある場合、記事作成・投稿の際にどのアカウントを使うか選択できます。

```
アカウントを切り替えて
```

```
/np:account switch
```

```
副業のたねアカウントで記事を書いて
```

## アカウント設定ファイルの場所

各アカウントは `accounts/{slug}.md` として管理されます。

```
note-pilot/
└── accounts/
    ├── account-template.md   ← テンプレート（削除しないでください）
    ├── thy-works.md          ← アカウント1
    ├── fuku-no-tane.md       ← アカウント2
    └── new-brand.md          ← アカウント3（preparing）
```
