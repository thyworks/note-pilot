# 使用イメージ

セットアップが完了したら、実際の使い方をイメージしてみましょう。note-pilot の典型的な流れを紹介します。

## 最初の一歩：自己紹介記事を書く

まず **自己紹介記事** を作ってみましょう。note.com では自己紹介が最初の記事として定番です。

```
自己紹介記事を書いて
```

Claude が記事を生成し、Gemini API キーが設定されていればアイキャッチ画像のスタイルを選ぶ画面が表示されます。スタイルを選んで簡単な調整を答えると画像が生成されます。その後、自動校閲 → note.com の下書き保存まで進みます。下書きが保存されたら note.com で内容を確認して公開しましょう。

公開したら URL を note-pilot に伝えておくと、次回から関連記事として自動でリンクされます。

```
記事を公開しました。URL は https://note.com/your-id/n/nxxxxxxxx です
```

## 連載計画を立てる

次に何を書くか迷ったときは連載計画機能を使います。

```
次のテーマを考えて
```

過去記事と重複しない3〜5案が提案されます。気に入ったテーマを選ぶと次回の記事作成に自動で反映されます。

詳しくは [連載計画を立てる](./plan) をご覧ください。

## 毎日の使い方

日常的な使い方はシンプルです。

```
記事を書いて
```

これだけで、アカウント設定のルールに従った記事を生成 → アイキャッチスタイル選択 → 自動校閲 → 下書き保存まで進みます。あとは note.com で確認して公開するだけです。

## 学んだことを蓄積する

記事を書いていくうちに「このタグは伸びた」「この書き方は読まれなかった」などの知見が生まれます。こうした学びをその都度 note-pilot に伝えると、次回以降の記事作成に活かされます。

```
「AI活用」タグがよく伸びるみたい。覚えておいて
```

詳しくは [ナレッジを追加する](./knowledge) をご覧ください。

## 全体の流れ

```
【初回】
自己紹介記事を書いて → 公開 → URL を登録

【毎回】
「記事を書いて」→ 下書き確認 → 公開 → URL を登録

【適宜】
「次のテーマを考えて」 → テーマ決定
「〇〇を覚えておいて」 → ナレッジ蓄積
```
