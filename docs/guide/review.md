# 校閲する

## 基本的な使い方

```
校閲して
```

```
チェックして
```

引数なしの場合は最新の下書きファイルを自動で探します。特定のファイルを指定することもできます。

```
この記事をレビューして
```

## チェックされる内容

30項目以上を自動チェックします。

### A. ファイル・メタデータ
- ファイル名の命名規則
- フロントマターの必須項目（created, tags, format など）
- タグ5個・一般向けタグの含有確認

### B. 貼り付けガイド
- ファイル冒頭のガイドブロックの存在と内容

### C. 本文構成
- タイトル30〜40字・【カテゴリタグ】の位置
- リード文の書き出し
- h2セクション数・各h2直前の区切り線
- CTA・注記・次回予告の存在

### D. 文章スタイル
- 総文字数2,500字以上
- 段落の長さ（3文以内）
- 比喩の使用有無
- 禁止要素（コードブロック・数式・パイプテーブル）の不在

### E. 内部リンク
- 関連記事リンク3〜5本
- URLが公開済みリストと一致するか

### F. アカウント整合性
- トーン・テーマの維持
- やらないことリストへの抵触確認

### G. 空行ルール
- 埋め込みカード・区切り線・箇条書きの前後の空行

## チェック後の処理

- **問題があった場合**: 修正内容を明示して記事ファイルを自動修正し、再チェックを行います。
- **全項目OK**: `✅ 全項目OK。投稿できます` と報告します。

チェック結果は `{review_dir}/YYYY-MM-DD-{スラッグ}-review.md` に保存されます。
