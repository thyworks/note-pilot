# 記事を書く — `/np:write`

## 基本的な使い方

Claude Codeに自然言語で話しかけます。

```
記事を書いて
```

```
今日の記事を書いて
```

```
動画生成AIについて記事を書いて
```

## 実行される処理

1. `accounts/` 内のアカウント設定ファイルを読み込む
2. `knowledge/writing-rules.md` のルールを確認する
3. `article_series.current_no` を参照して次の連番を決定する
4. `templates/article.md` をベースに記事を生成する
5. `{output_dir}/{slug}-article-{連番3桁}-note-paste.md` として保存する
6. アイキャッチ画像を自動生成する（`GEMINI_API_KEY` 設定済みの場合のみ）
7. 自動的に `/np:review` を呼び出してチェックを実施する

## 記事のルール（自動適用）

note-pilotは以下のルールを自動で守ります。

### タイトル
- 30〜40字
- 先頭に【カテゴリタグ】をつける（例: 【AI入門】）

### 本文構成
- リード文は問いかけまたは共感から始める
- リード直後に自己紹介埋め込みを配置
- h2セクションを3〜5個設ける
- 各h2の直前に区切り線（---）を入れる
- 1段落は3文以内
- 比喩を1個以上使う

### 末尾
- フォロー・スキのCTA
- em囲みの注記
- 次回予告（`next_topics` から選択）
- 関連記事リンク（3〜5本）

### 禁止事項
- コードブロック・数式の使用
- markdownリンク記法 `[text](url)` の使用
- パイプ記法テーブル `|col|col|`
- アカウント設定の「やらないこと」リストに触れる内容

## 複数アカウントがある場合

`accounts/` フォルダに複数の設定ファイルがある場合、どのアカウントで書くかを聞かれます。

```
どのアカウントで書きますか？
1. my-account-a（記事#015まで作成済み）
2. my-account-b（記事#003まで作成済み）
```

番号で選択すると、そのアカウント設定で記事が生成されます。
